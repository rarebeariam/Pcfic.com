// capabilities

What makes PCF different

The industry spent 30 years building detectors that learn what attacks look like. PCF asks a different question: how does attack behavior differ structurally from normal behavior?

// feature_01

Structural, not statistical learning

PCF computes Positional Signature Vectors using PMI (Pointwise Mutual Information) across behavioral token positions. The result is a geometric fingerprint — deterministic, auditable, reproducible.

zero_training_samples: true
// feature_02

No retraining when attacks mutate

Adversarial mutation changes packet content, timing, and headers. It does not change the structural geometry of attack behavior. PSV divergence persists regardless of variant — no model update required.

observation_window: 0_days
// feature_03

CPU-native, inline-capable

PCF is pure numerical computation. ~1.5ms per trace on commodity hardware. No inference server, no model loading, no GPU cluster. Deploys inline between your network tap and your SIEM.

// how_it_works

PCF detection pipeline

Four deterministic steps. No black box. Every detection is traceable back to structural measurements in the PSV — auditable by design.

step_01 // tokenize

Network events to behavioral tokens

Raw packet flows mapped to semantic tokens: AUTH_BRUTE, NET_FLOOD, NET_PORTSCAN, NET_SLOW_CONN, NET_LARGE_SEND, NET_CONNECT, SYS_READ — capturing behavioral fingerprints.

step_02 // vectorize

Compute Positional Signature Vector

PCF structural analysis applied over token co-occurrence at position offset k. (Patent Pending)

step_03 // classify

Dual-signal structural comparison

Signal A: PSV cosine distance vs benign centroid exceeds 90th percentile threshold. Signal B: attack-token fraction exceeds 99th percentile. OR logic — either fires an alert.

step_04 // alert

Structured alert with provenance

Alert includes token sequence, PSV vector, cosine distance, signal triggers, and latency. No black-box score — full structural provenance included in every detection event.
